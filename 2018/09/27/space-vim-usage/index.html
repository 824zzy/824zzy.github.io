从Pycharm到SpaceVim

Posted on 2018-09-27 | Post modified: 2018-11-25 | In Linux | Visitors:
Words count in article: 316 | Reading time ≈ 1分钟

Pycharm太吃内存，因此转向使用SpaceVim。基本配置记录如下：

环境基本配置

安装SpaceVim

配置.SpaceVim/autoload/SpaceVim.vim

  • set timeoutlen=10:空格键延迟
  • let g:spacevim_relativenumber=0:禁用相对行号

配置.SpaceVim.d/init.vim

  • [[layers]] \n name="lang#python":使用python
  • disable_plugins=["neomake.vim"]:禁用插件（当使用conda的时候，这个插件会报错）

更新:SPUpdate

常用命令备忘

基本操作

  • e: 打开一个空的编辑器
  • :e file_name: 打开文件（该操作也可以在nerdtree上完成）

space+b系列(缓冲区)

  • N+l:在右侧新建buffer
  • d:删除buffer

space+f系列(文件管理)

  • t:开关nerdtree

space+w系列(窗口管理)

  • o:切换到下一个窗口
  • /和-:分别在右侧和下方分隔窗口
  • F: 新建Tab

space+t系列（Toggle管理）

  • t:打开tab管理

space+c系列(注释）

  • l:注释选中行（配合V模式）

space+s系列(搜索)

  • s:直接在当前文件搜索

space+l系列（语言）

  • r:执行代码

参考与引用

  • https://spacevim.org/cn/layers
  • https://everettjf.gitbooks.io/spacevimtutorial
